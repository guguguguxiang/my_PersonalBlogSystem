# 个人博客系统 V1.0 - 产品需求文档 (PRD)

## 1. 产品概述
本项目是一个基于 React + Node.js + MySQL 的前后端分离架构个人博客系统。主要用于作者分享技术文章、沉淀知识，同时向访客展示个人技术能力。V1.0 版本聚焦于核心的内容发布与展示闭环。

## 2. 核心功能详述

### 2.1 用户模块 (User Module)
* **注册 (Register)：**
    * 所需字段：用户名（3-20位）、邮箱（需校验格式）、密码（至少 6 位，需二次确认）。
    * 规则：用户名和邮箱必须全站唯一。密码在数据库中必须加密存储（如 bcrypt）。
* **登录 (Login)：**
    * 支持使用 用户名 或 邮箱 登录。
    * 认证机制：登录成功后下发 JWT Token，前端存储在 localStorage 或 Cookie 中。Token 有效期建议设为 7 天。
* **个人中心 (Profile)：**
    * 权限：需携带有效 Token 访问。
    * 功能：查看个人基础信息，支持修改头像 URL、修改密码。

### 2.2 文章模块 (Post Module)
* **发布文章 (Create Post)：**
    * 权限：仅限已登录用户（作者）操作。
    * 内容输入：标题（必填，最多 100 字）、文章内容（支持 Markdown 语法，必填）、分类（单选，必填）、标签（多选，非必填，最多 5 个）、封面图（URL 输入或上传，非必填）。
    * 状态管理：支持直接“发布”或存为“草稿”。
* **编辑文章 (Edit Post)：**
    * 权限：只能编辑自己发布的文章。
    * 功能：回显文章所有已有数据，保存后更新 `updated_at` 时间。
* **删除文章 (Delete Post)：**
    * 权限：只能删除自己的文章。
    * 交互：点击删除需有二次确认弹窗（防误触）。
* **查看文章详情 (View Post)：**
    * 权限：公开访问。
    * 展示内容：标题、作者信息、发布时间、分类、标签、正文（将 Markdown 渲染为 HTML）、浏览次数。
    * 规则：每次有效访问，数据库中的 `views` 浏览次数自动 +1。
* **文章列表查询 (List Posts)：**
    * 权限：公开访问。
    * 分页逻辑：支持基于 `page` 和 `pageSize` 的分页拉取（默认每页 10 条）。
    * 筛选与检索：支持按“分类 ID”筛选，支持按“关键词”（匹配标题或内容）模糊搜索。按创建时间倒序排列。

## 3. 前端页面与路由规划

* **`/` (首页)：** 顶部导航栏 + 包含搜索/分类筛选的文章列表组件 + 侧边栏（作者简介/热门标签）。
* **`/post/:id` (文章详情页)：** 核心阅读区，需具备良好的排版样式（如 Typography）。
* **`/login` & `/register` (登录/注册页)：** 居中表单，包含基础的格式校验提示。
* **`/write` (创作页)：** 鉴权路由。左侧 Markdown 源码编辑区，右侧实时 HTML 预览区。
* **`/profile` (个人中心)：** 鉴权路由。展示和修改个人设置，下方可列出自己名下的文章列表（包含编辑/删除操作入口）。

## 4. 非功能性需求 (NFR)
* **响应式设计：** 前端页面需兼容 PC 端与主流移动端设备的显示（利用 Tailwind CSS 的响应式断点）。
* **安全性：** 所有写操作（增、删、改）API 必须进行 Token 校验；防范基础的 SQL 注入和 XSS 攻击。
